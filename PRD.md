## 1. 产品定位与愿景

### 1.1 产品名称

**ffeel** — "Fast File Exchange over Ethernet/LAN" 的缩写，传递"让局域网文件交换像呼吸一样自然"的理念。

### 1.2 产品定位

ffeel 是一款开源的局域网文件共享桌面工具，专注于在**同一局域网**内的设备之间提供**高速、安全、零配置**的文件传输体验。无需互联网连接、无需账号注册、无需云服务器中转。

### 1.3 核心价值主张

| 维度 | 价值 |
|------|------|
| **零配置** | 安装即用，mDNS 自动发现设备，无需 IP 输入或账号注册 |
| **高速传输** | 直连局域网，利用 HTTP/2 流式传输，速度仅受限于网络带宽 |
| **安全可控** | TLS 加密通信 + 设备配对认证 + 目录沙箱，数据不经过第三方 |
| **跨平台** | 基于 Tauri 2.0 + Rust，原生支持 Windows、macOS、Linux |
| **传输管理** | 队列管理 + 暂停/恢复 + 进度监控，支持大文件断点续传 |

### 1.4 与竞品差异化

| 竞品 | ffeel 优势 | ffeel 不足 |
|------|-----------|-----------|
| **AirDrop** (Apple 生态) | 跨平台支持 (Win/Linux/macOS)；不依赖蓝牙+WiFi 组合 | 无 AirDrop 的近距离感知；无系统级集成 |
| **SMB/CIFS** (文件共享) | 安装即用无需配置共享；自动发现设备；图形化界面；传输队列管理 | 不适合长期挂载的网络存储场景 |
| **SendAnywhere** | 完全离线，不依赖中转服务器；无文件大小限制；不限传输次数 | 无法跨局域网传输；无 Web 链路分享 |
| **SCP/rsync** (命令行) | 图形化界面；自动设备发现；拖拽操作；传输进度可视化 | 无脚本化/自动化能力 |
| **LAN Share / Dukto** | 正在开发中更完整的安全模型 (TLS+配对)；Rust 性能优势 | 起步阶段，生态较小 |

## 2. 目标用户

### 2.1 用户画像

#### 画像 A: 跨平台办公人员 — "小明"

| 属性 | 描述 |
|------|------|
| 背景 | 25-35 岁，软件工程师/设计师，同时在用 macOS 笔记本和 Windows 台式机 |
| 痛点 | 在两个系统间传文件需要 U 盘或网盘中转，流程繁琐且有隐私风险 |

| 技术能力 | 中高，能理解简单的配置 |
| 核心需求 | 便捷的跨平台传输，拖拽即传，传输历史可追踪 |

#### 画像 B: 家庭/宿舍用户 — "小张"

| 属性 | 描述 |
|------|------|

| 使用方式 | 低频但传输量大（视频、照片集），一次可能传数 GB |
| 技术能力 | 低，期望"打开就能用" |
| 核心需求 | 简单直观，需要文件夹传输和断点续传 |

#### 画像 C: 企业内网 IT 管理员 — "老王"

| 属性 | 描述 |
|------|------|
| 背景 | 企业内部 IT 支持，需在隔离内网中向多台机器分发文件 |
| 痛点 | 内网无互联网，传统的 USB 拷贝效率低；需要审计传输记录 |
| 使用方式 | 中频，批量分发安装包、更新补丁 |
| 技术能力 | 高，关注安全性和可配置性 |
| 核心需求 | 设备配对认证、传输加密、日志记录、限速控制 |

## 3. 功能规格

### 3.1 设备发现

#### F-001: 自动设备发现

| 项目 | 描述 |
|------|------|
| **功能描述** | 应用启动后自动通过 mDNS 扫描局域网，发现运行 ffeel 的其他设备 |
| **前置条件** | 设备处于同一局域网子网；目标设备已启动 ffeel 并注册了 mDNS 服务 |
| **操作流程** | 1. 应用启动时自动调用 `start_discovery`<br>2. 后端通过 `mdns-sd` 库广播 `_ffeel._tcp.local.` 服务类型<br>3. 监听到 `ServiceResolved` 事件后构建 `DeviceInfo`<br>4. 通过 Tauri 事件 `device-found` 推送到前端 |
| **预期结果** | 设备列表自动显示可用的远程设备，每台设备显示名称、IP、端口、平台 |
| **异常处理** | mDNS 初始化失败时在日志中记录错误，前端显示提示"设备发现服务启动失败"<br>无设备的局域网显示空状态和引导提示 |

#### F-002: 手动扫描

| 项目 | 描述 |
|------|------|
| **功能描述** | 用户点击"刷新"按钮强制重新扫描局域网设备 |
| **前置条件** | 应用已正常运行 |
| **操作流程** | 1. 用户点击"刷新"按钮<br>2. 前端调用 `startDiscovery`<br>3. 后端重新启动 mDNS 浏览 (Browse)<br>4. 设备事件通过 broadcast channel 推送 |
| **预期结果** | 设备列表刷新，新上线的设备出现，已离线的设备标记为 offline |
| **异常处理** | 同 F-001 |

#### F-003: 设备状态跟踪

| 项目 | 描述 |
|------|------|
| **功能描述** | 跟踪设备在线/离线状态，当设备离开局域网时及时更新 |
| **前置条件** | 已发现至少一台设备 |
| **操作流程** | 后端监听到 `ServiceRemoved` 事件 → 通过 `device-lost` 事件推送 → 前端更新设备状态或从列表移除 |
| **预期结果** | 离线设备卡片的在线指示点变为灰色 |
| **异常处理** | mDNS 事件可能延迟，允许 2 次心跳超时后才标记离线 |

#### 边界条件

- **同一设备多个网络接口**: 取第一个非 loopback 的 IPv4 地址
- **设备名冲突**: mDNS 自动处理命名冲突，应用层不做额外处理
- **广播风暴防护**: 扫描间隔不低于 3 秒
- **NAT/跨子网**: 不支持，仅在同一广播域内工作

### 3.2 文件浏览

#### F-004: 目录树浏览

| 项目 | 描述 |
|------|------|
| **功能描述** | 点击远程设备后进入文件浏览器，以目录层级方式浏览共享目录 |
| **前置条件** | 已选择一台在线远程设备 |
| **操作流程** | 1. 前端调用 `browseDirectory(deviceIp, port, "")`<br>2. 后端发送 GET `/api/list?path=` 到远程设备<br>3. 远程设备返回目录列表 (DirEntry 数组)<br>4. 前端以表格形式展示文件和文件夹<br>5. 双击文件夹进入下层目录；点击"上级目录"按钮返回 |
| **预期结果** | 文件列表正确展示，目录排在文件前，文件夹可导航进入 |
| **异常处理** | HTTP 请求超时 (10s) → 显示"连接远程设备失败"<br>路径不存在 → 显示"路径不存在" |

#### F-005: 文件排序

| 项目 | 描述 |
|------|------|
| **功能描述** | 文件列表按目录优先、名称字母序排序 |
| **前置条件** | 目录列表已加载 |
| **操作流程** | 后端在 `list_directory` 中对结果排序：先目录后文件，同类型按名称字典序 |
| **预期结果** | 目录始终排列在文件上方，名称有序排列 |
| **异常处理** | N/A (后端排序逻辑，无用户交互失败点) |

#### F-006: 文件名搜索 (未来版本)

| 项目 | 描述 |
|------|------|
| **功能描述** | 在当前目录中按文件名关键词模糊搜索 |
| **前置条件** | 至少加载了一个目录；当前目录条目数大于 10 |
| **操作流程** | 前端输入关键词 → 发送搜索请求 → 后端过滤匹配的 DirEntry → 返回结果 |
| **预期结果** | 实时筛选出文件名包含关键词的文件和文件夹 |
| **异常处理** | 搜索超时或失败时回退到显示全量列表 |

#### 边界条件

- **空目录**: 显示"目录为空"空状态
- **访问越权路径**: `resolve_safe_path` 阻止目录遍历攻击，返回 403
- **符号链接**: 默认不跟随符号链接 (避免循环引用)
- **文件路径含特殊字符**: URL 编码处理 (`urlencoding::encode`)
- **超大目录 (10K+ 文件)**: 懒加载，暂不分页 (后续可加)

### 3.3 文件传输

#### F-007: 单文件下载

| 项目 | 描述 |
|------|------|
| **功能描述** | 从远程设备下载单个文件到本地 |
| **前置条件** | 文件浏览目录已加载；目标文件不为目录 |
| **操作流程** | 1. 用户在文件列表中点击某文件的"下载"按钮<br>2. 前端调用 `downloadFile(deviceIp, port, remotePath, localPath, fileName, fileSize)`<br>3. 后端创建 TransferTask (状态 Pending)→ 加入队列 → 异步启动下载<br>4. 后端 HTTP GET `/api/download?path=<remotePath>` 到远程设备<br>5. 流式接收响应 Body 并写入本地文件<br>6. 定期更新 `bytes_transferred` 和 `speed` |
| **预期结果** | 文件完整下载到本地，传输管理中出现完成记录 |
| **异常处理** | 远程文件不存在 → 404 错误，任务标记 Failed<br>网络中断 → 任务标记 Failed (后续支持断点续传)<br>磁盘空间不足 → 任务标记 Failed |

#### F-008: 多文件/批量下载

| 项目 | 描述 |
|------|------|
| **功能描述** | 用户勾选多个文件后批量添加到下载队列 |
| **前置条件** | 目录中有至少 2 个非目录文件 |
| **操作流程** | 1. 用户勾选文件复选框 → 出现批量操作栏<br>2. 点击"下载选中" → 为每个文件创建独立的 TransferTask<br>3. 各任务按顺序加入传输队列，并发数受 `max_concurrent_transfers` 限制 |
| **预期结果** | 每个文件作为一个独立任务依次传输，同时进行中的任务不超过并发限制 |
| **异常处理** | 部分文件失败不影响其他文件的传输 |

#### F-009: 文件夹传输 (未来版本)

| 项目 | 描述 |
|------|------|
| **功能描述** | 传输整个文件夹及其子目录内容 |
| **前置条件** | 远程目录存在且有读取权限 |
| **操作流程** | 1. 用户点击文件夹的"下载"按钮<br>2. 后端递归遍历远程目录结构<br>3. 为目录中所有文件创建传输任务<br>4. 保留目录层级结构在本地 |
| **预期结果** | 文件夹完整复制到本地，包含所有子目录和文件 |
| **异常处理** | 目录遍历中断 → 已有文件保留，报告未完成列表 |

#### F-010: 拖拽下载 (未来版本)

| 项目 | 描述 |
|------|------|
| **功能描述** | 用户将文件浏览器中的条目拖拽到系统文件夹，触发下载 |
| **前置条件** | 操作系统支持拖拽操作 |
| **操作流程** | 1. 用户选中文件/文件夹，拖拽到应用外的文件管理器窗口<br>2. Tauri 拖拽事件捕获目标路径<br>3. 自动创建下载任务 |
| **预期结果** | 文件下载到用户拖拽的目标位置 |
| **异常处理** | 拖拽目标不可写 → 提示错误 |

#### F-011: 文件上传 (未来版本)

| 项目 | 描述 |
|------|------|
| **功能描述** | 将本地文件上传到远程设备的共享目录 |
| **前置条件** | 远程设备在线；当前目录可写 |
| **操作流程** | 1. 用户点击"上传"或拖拽本地文件到文件浏览器<br>2. 前端显示文件选择对话框 (Tauri dialog plugin)<br>3. 选择文件后 → 调用上传 API<br>4. 后端 HTTP POST multipart/form-data 到远程 `/api/upload`<br>5. 远程设备流式写入共享目录 |
| **预期结果** | 文件出现在远程设备的当前目录中 |
| **异常处理** | 上传中断 → 部分文件保留，任务标记 Failed<br>目标目录无写入权限 → 403 错误 |

#### 边界条件

- **空文件**: 正常传输，进度立即 100%
- **超大文件 (>4GB)**: 使用流式传输，不加载到内存；文件大小字段使用 `u64` 支持到 16EB
- **文件名冲突**: 远程下载到本地时如果文件已存在，自动添加 `(1)`, `(2)` 后缀
- **并发上限**: `max_concurrent_transfers` 默认 3，可在设置中调整 1-10

### 3.4 传输管理

#### F-012: 传输队列

| 项目 | 描述 |
|------|------|
| **功能描述** | 所有传输任务在统一队列中管理，分为"正在传输"、"已暂停"、"历史记录"三个分组 |
| **前置条件** | 至少创建了一个传输任务 |
| **操作流程** | 1. 用户切换到"传输"标签页<br>2. 前端调用 `getTransfers()`<br>3. 后端返回所有 TransferTask 列表<br>4. 前端按状态分组显示 |
| **预期结果** | 任务按状态清晰分组，每组有数量标记 |
| **异常处理** | N/A |

#### F-013: 传输进度展示

| 项目 | 描述 |
|------|------|
| **功能描述** | 每个传输任务显示进度条、已传输/总大小、速度、状态标签 |
| **前置条件** | 至少有一个非完成状态的任务 |
| **操作流程** | 后端在 `download_file_from_remote` 中定期 (每次 chunk) 更新 `bytes_transferred` 和 `speed` → 前端每 2 秒轮询 `getTransfers` 刷新显示 |
| **预期结果** | 进度条平滑增长，速度显示实时变化 |
| **异常处理** | 速度计算使用 `downloaded / elapsed`，避免除零错误 |

#### F-014: 暂停/继续传输

| 项目 | 描述 |
|------|------|
| **功能描述** | 用户主动暂停正在进行的传输；已暂停的传输可继续 |
| **前置条件** | 任务状态为 Transferring (可暂停) 或 Paused (可继续) |
| **操作流程** | **暂停**: 用户点击"暂停" → 调用 `pauseTransfer(id)` → 后端设置状态为 Paused → 下载循环检测到 Paused 状态后停止<br>**继续**: 用户点击"继续" → 调用 `resumeTransfer(id)` → 后端设置状态为 Pending → 队列调度器重新拉起 |
| **预期结果** | 传输立即停止 (下次 chunk 写入前检查)；继续后从断点重传 |
| **异常处理** | 尝试暂停非活跃任务 → 错误 "只能暂停正在进行的传输"<br>尝试恢复非暂停任务 → 错误 "只能恢复已暂停的传输" |

#### F-015: 取消传输

| 项目 | 描述 |
|------|------|
| **功能描述** | 用户取消一个待处理/进行中/已暂停的传输任务 |
| **前置条件** | 任务状态为 Pending / Transferring / Paused |
| **操作流程** | 用户点击"取消" → 调用 `cancelTransfer(id)` → 后端设置状态为 Cancelled → 部分下载文件默认保留 (不自动清理) |
| **预期结果** | 任务标记为 Cancelled，不再继续传输 |
| **异常处理** | 已完成/已失败/已取消的任务不显示取消按钮 |

#### F-016: 断点续传 (未来版本)

| 项目 | 描述 |
|------|------|
| **功能描述** | 传输中断后从上次中断位置继续传输，无需重新开始 |
| **前置条件** | 传输中断但部分文件已保存到本地；远程服务器支持 Range 请求 |
| **操作流程** | 1. 下载端检测到本地已有部分文件 → 读取已下载字节数<br>2. 向远程发送 HTTP Range 请求: `Range: bytes=<downloaded>-`<br>3. 远程返回 206 Partial Content 和剩余部分流<br>4. 以追加模式写入文件 |
| **预期结果** | 仅传输未完成的部分，节省带宽和时间 |
| **异常处理** | 远程不支持 Range → 回退到完整重传<br>本地文件校验失败 → 完整重传 |

#### F-017: 传输限速 (未来版本)

| 项目 | 描述 |
|------|------|
| **功能描述** | 根据用户在设置中配置的 `speed_limit` 控制传输速度 |
| **前置条件** | `speed_limit` > 0 |
| **操作流程** | 每个 chunk 写入后计算当前速度，若超过限制则添加延时 |
| **预期结果** | 传输速度不高于设置的限速值 |
| **异常处理** | 限速值修改后对新的传输任务立即生效；已有任务可配置是否立即应用 |

#### 边界条件

- **队列满**: 达到 `max_concurrent_transfers` 并发限制时，新任务自动排入 Pending 等待
- **传输历史上限**: 保留最近 100 条完成/失败/取消记录，超出自动清理最旧的
- **应用重启**: 当前传输队列保存在内存中，重启后丢失 (后续可加持久化)

### 3.5 安全

#### F-018: TLS 传输加密 (未来版本)

| 项目 | 描述 |
|------|------|
| **功能描述** | 设备间通信使用 TLS 1.3 加密，防止局域网嗅探 |
| **前置条件** | 设备需生成/交换自签名证书 |
| **操作流程** | 1. 首次运行时生成本机自签名证书和私钥<br>2. HTTP 服务升级为 HTTPS (axum 支持 TLS)<br>3. 设备发现时交换证书指纹<br>4. 通信时验证对方证书指纹 |
| **预期结果** | 所有 HTTP/WebSocket 通信经过 TLS 加密 |
| **异常处理** | 证书指纹不匹配 → 连接拒绝，提示用户确认 |

#### F-019: 设备配对认证

| 项目 | 描述 |
|------|------|
| **功能描述** | 通过配对码验证新设备的身份，防止未授权访问 |
| **前置条件** | 设置中启用 `require_pairing` |
| **操作流程** | 1. 设备 A 首次尝试连接设备 B<br>2. 设备 B 显示 4 位配对码<br>3. 用户在设备 A 上输入配对码<br>4. 设备 B 验证配对码 → 配对成功后加入 `trusted_devices` 列表 |
| **预期结果** | 未配对设备无法浏览或传输文件 |
| **异常处理** | 配对码错误 → 提示"配对码错误，请重试"<br>配对码过期 (5 分钟) → 自动生成新码 |

#### F-020: 目录沙箱

| 项目 | 描述 |
|------|------|
| **功能描述** | 严格限制文件访问范围不超过设定的共享目录 |
| **前置条件** | 需设置共享目录 |
| **操作流程** | 每个文件操作路径都经过 `resolve_safe_path` 校验：将请求路径与共享目录分别做 `canonicalize()` 后检查前缀匹配 |
| **预期结果** | 拒绝所有尝试访问共享目录之外路径的请求 |
| **异常处理** | 路径越界 → 返回 403 "访问被拒绝：路径超出共享目录范围"<br>路径不存在 → 返回 404 |

#### F-021: 信任设备管理

| 项目 | 描述 |
|------|------|
| **功能描述** | 在设置中查看和管理已配对的信任设备列表 |
| **前置条件** | 至少完成一次设备配对 |
| **操作流程** | 用户进入设置 → 查看 `trusted_devices` 列表 → 可移除某个信任设备 |
| **预期结果** | 被移除的设备下次连接需重新配对 |
| **异常处理** | N/A |

#### 边界条件

- **自签名证书信任**: 同一设备首次连接时需手动确认证书指纹
- **MITM 防护**: 证书指纹 + 配对码双重验证
- **设置关闭认证**: 关闭 `require_pairing` 后，局域网内所有 ffeel 设备可自由连接

### 3.6 设置

#### F-022: 共享目录配置

| 项目 | 描述 |
|------|------|

| **前置条件** | 路径存在且可读 |
| **操作流程** | 用户在设置页输入或浏览选择目录路径 → 点击"保存" → 调用 `updateSettings` |

#### F-023: 设备名称配置

| **功能描述** | 用户可自定义本机在局域网中显示的名称 |

| **操作流程** | 设备名默认取系统 hostname → 用户在设置页修改 → 保存 → 重启 mDNS 服务广播新名称 |
| **预期结果** | 其他设备看到的名称更新为新名称 |

#### F-024: 并发传输数配置

| **前置条件** | N/A |
| **操作流程** | 数字输入 (1-10) → 保存 → 实时生效 |
| **预期结果** | 活跃传输任务数不超过设置值 |
| **异常处理** | 输入 < 1 → 限制为 1；输入 > 10 → 限制为 10 |

#### F-025: 传输速度限制

| **预期结果** | 所有传输任务的总速度不超过限速值 |
| **异常处理** | 输入负数 → 视为 0 (不限速) |

#### F-026: 配对认证开关

| 项目 | 描述 |
|------|------|

## 4. 用户流程

### 4.1 主流程：发现 → 浏览 → 传输

```
[应用启动] 
    │
    ├─→ 自动设备发现 (mDNS)
    │      │
    │      ├─→ [发现设备] → 设备列表展示
    │      │                    │
    │      │              [点击设备卡片]
    │      │                    │
    │      │              [进入文件浏览]
    │      │                    │
    │      │              [导航目录/选择文件]
    │      │                    │
    │      │         ┌──────────┴──────────┐
    │      │    [单文件下载]          [批量下载]
    │      │         │                    │
    │      │    [任务入队]            [依次入队]
    │      │         │                    │
    │      │    [切换传输标签页查看进度]
    │      │
    │      └─→ [无设备] → 显示空状态引导
    │                        │
    │                   [点击刷新重试]
    │
    └── [mDNS 失败] → 日志记录，提示用户
```

### 4.2 设备配对流程

```
[设备 A 尝试连接设备 B]
    │
    ├─→ [设备 B 已启用配对认证]
    │       │
    │       ├─→ [设备 B 是信任设备]
    │       │       ↓
    │       │   跳过配对，直接建立连接
    │       │
    │       └─→ [设备 B 是新设备]
    │               │
    │               ├─→ [设备 B 生成 4 位配对码并显示在屏幕上]
    │               │
    │               ├─→ [设备 A 弹出配对码输入框]
    │               │
    │               ├─→ [用户在设备 A 输入配对码]
    │               │       │
    │               │       ├─→ [配对码正确]
    │               │       │       ↓
    │               │       │   设备 B 将设备 A 加入 trusted_devices
    │               │       │   连接建立
    │               │       │
    │               │       └─→ [配对码错误]
    │               │               ↓
    │               │           提示错误，允许重试
    │               │
    │               └─→ [配对码超时 (5 分钟)]
    │                       ↓
    │                   自动生成新配对码
    │
    └─→ [设备 B 未启用配对认证]
            ↓
        直接建立连接
```

### 4.3 传输取消/重试流程

```
[正在进行/已暂停的传输任务]
    │
    ├─→ [用户点击"取消"]
    │       │
    │       ├─→ [正在传输中]
    │       │       ↓
    │       │   停止下载循环 (下次 chunk 检查)
    │       │   标记状态为 Cancelled
    │       │   保留部分下载的文件
    │       │
    │       ├─→ [已暂停]
    │       │       ↓
    │       │   标记状态为 Cancelled
    │       │
    │       └─→ [等待中]
    │               ↓
    │           标记状态为 Cancelled
    │
    └─→ [传输失败]
            │
            └─→ [用户重新下载]
                    ↓
                创建新任务 (非恢复旧任务)
                重新进入队列
```

### 4.4 断点续传流程 (未来版本)

```
[网络中断导致传输失败]
    │
    ├─→ [传输管理器检测到连接断开]
    │       │
    │       ├─→ 标记任务为 Failed
    │       │   记录已下载字节数
    │       │
    │       └─→ [用户点击"继续"]
    │               │
    │               ├─→ [检测到本地存在部分文件]
    │               │       │
    │               │       ├─→ [远程服务器支持 Range 请求]
    │               │       │       ↓
    │               │       │   HEAD 请求获取远程文件大小
    │               │       │   发送 `Range: bytes={downloaded}-` 请求
    │               │       │   接收 206 Partial Content
    │               │       │   以追加模式继续写入
    │               │       │   合并进度显示
    │               │       │
    │               │       └─→ [远程不支持 Range]
    │               │               ↓
    │               │           完整重传
    │               │
    │               └─→ [无部分文件]
    │                       ↓
    │                   完整重传
    │
    └── [应用重启后恢复]
            ↓
        (未来版本) 读取传输记录持久化文件
        检测是否存在未完成的 .ffeel.part 文件
        支持续传
```

### 4.5 设置修改流程

```
[用户进入设置页]
    │
    ├─→ 加载当前设置 (get_settings)
    │
    ├─→ 用户修改: [设备名称 | 共享目录 | 并发数 | 限速 | 配对开关]

```

## 5. UI/UX 设计原则

### 5.1 设计风格

- **主题**: 暗色主题，深蓝/黑色背景 (`#1a1a2e` 主背景, `#16213e` 二级背景)

- **圆角**: 统一 8px 圆角 (`--radius: 8px`)
- **阴影**: 轻微阴影提升层次感 (`0 2px 8px rgba(0,0,0,0.3)`)

### 6.4 通信协议

| 协议 | 用途 | 详情 |
|------|------|------|
| **mDNS** (multicast DNS) | 设备发现 | 服务类型 `_ffeel._tcp.local.`；TXT 记录携带 `id`, `name`, `platform`；库: `mdns-sd` |
| **HTTP/1.1** (axum) | 文件浏览/下载/上传 | 无状态 REST 风格 API；所有路径参数经 `resolve_safe_path` 校验 |
| **WebSocket** (axum) | 实时事件推送 | (当前为骨架) 心跳间隔 5s；未来用于实时进度推送以替代轮询 |
| **Tauri IPC** | 前后端通信 | Command 模式 (invoke) 用于请求-响应；Event 模式 (emit/listen) 用于推送事件 |

### 6.5 安全模型

```
                  ┌─────────────────────────┐
                  │   ffeel Security Model    │
                  └─────────────────────────┘
                           │
          ┌────────────────┼────────────────┐
          │                │                │
          ▼                ▼                ▼
    ┌──────────┐    ┌──────────┐    ┌──────────────┐
    │ 传输加密  │    │ 设备认证  │    │ 访问控制     │
    │ (TLS 1.3)│    │ (配对码)  │    │ (目录沙箱)   │
    └──────────┘    └──────────┘    └──────────────┘
         │               │                │
         ▼               ▼                ▼
    • 自签名证书    • 4 位配对码      • canonicalize()
    • 证书指纹交换  • SHA256 哈希验证 • starts_with 前缀检查
    • 可选启用      • trusted_devices  • 路径越界 → 403
                    • 5 分钟超时
```

**分层安全策略**:

1. **网络层**: TLS 加密 (未来版本)，防止中间人窃听
2. **身份层**: 配对码验证信任设备，防止未授权访问
3. **资源层**: 目录沙箱路径校验，防止目录遍历攻击
4. **传输层**: (未来) 文件校验和验证

### 7.2 HTTP 端点 (设备间通信)

所有端点运行在 axum HTTP 服务器上，端口随机分配 (绑定 `0.0.0.0:0`)。

#### H-01: GET /api/list

| 项目 | 描述 |
|------|------|
| **描述** | 列出共享目录中指定路径的文件和目录 |
| **参数** | `path` (query, optional): 相对共享目录的路径，空=根目录 |
| **返回** | `200 OK`: JSON 数组 `[DirEntry, ...]`<br>`400 Bad Request`: 路径不是目录<br>`403 Forbidden`: 路径超出共享目录<br>`404 Not Found`: 路径不存在 |
| **安全性** | `resolve_safe_path` 校验，阻止目录遍历 |

#### H-02: GET /api/download

| 项目 | 描述 |
|------|------|
| **描述** | 流式下载文件 |
| **参数** | `path` (query, required): 相对共享目录的文件路径 |
| **返回** | `200 OK`: 文件流，包含 `Content-Type` 和 `Content-Disposition` 头部<br>`400 Bad Request`: 路径是目录/无效<br>`403 Forbidden`: 路径越界<br>`404 Not Found`: 文件不存在 |
| **安全性** | `resolve_safe_path` 校验；使用 `mime_guess` 自动检测 MIME 类型 |

#### H-03: POST /api/upload

| 项目 | 描述 |
|------|------|
| **描述** | 上传文件到共享目录 |
| **参数** | `path` (query, optional): 目标子目录路径<br>Body: multipart/form-data, field name "file" |
| **返回** | `200 OK`: `{"saved_bytes": <number>}`<br>`400 Bad Request`: 请求格式错误 |
| **安全性** | `resolve_safe_path` 校验目标目录；流式写入避免内存溢出 |

#### H-04: GET /api/health

| 项目 | 描述 |
|------|------|
| **描述** | 健康检查端点 |
| **参数** | 无 |
| **返回** | `200 OK`: `{"status": "ok"}` |

pub struct DeviceInfo {

### 8.2 DirEntry

```rust
pub struct DirEntry {
    pub name: String,           // 文件/目录名
    #[serde(rename = "type")]
    pub entry_type: String,     // "file" | "directory"
    pub size: Option<u64>,      // 文件大小 (字节)，目录为 null
    pub modified_at: Option<String>,  // 修改时间 (Unix 时间戳)
    pub is_dir: bool,           // 是否为目录

```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `name` | String | 非空，有效文件名字符 | 不含路径的分隔符 |

| `is_dir` | bool | - | 主排序键 (目录优先) |

### 8.3 TransferTask

```rust
pub struct TransferTask {
    pub id: String,              // 任务唯一 ID (如 "transfer_1")
    pub file_name: String,       // 文件名
    pub file_size: u64,          // 文件总大小 (字节)
    pub bytes_transferred: u64,  // 已传输字节数
    pub status: TransferStatus,  // 任务状态
    pub direction: TransferDirection,  // 传输方向
    pub remote_device: String,   // 远程设备标识 (IP:port)
    pub remote_path: String,     // 远程相对路径
    pub local_path: String,      // 本地绝对路径
    pub speed: f64,              // 当前传输速度 (bytes/sec)
    pub error: Option<String>,   // 错误信息 (仅在 Failed 时有值)
    pub created_at: String,      // RFC3339 时间戳
}
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | String | 格式 `transfer_{n}` | 全局递增，不重复 |
| `file_name` | String | 非空 | 仅文件名，不含路径 |
| `file_size` | u64 | >= 0 | 远程文件大小 |
| `bytes_transferred` | u64 | 0 <= x <= file_size | 实时更新 |
| `status` | TransferStatus | 见下方枚举 | 状态机流转 |
| `direction` | TransferDirection | Download / Upload | 区分方向 |
| `remote_device` | String | "ip:port" | 远程设备地址 |
| `remote_path` | String | 相对共享目录的路径 | 不含前导 `/` |
| `local_path` | String | 系统合法绝对路径 | 下载目标位置 |
| `speed` | f64 | >= 0.0 | 每 chunk 更新 |
| `error` | Option\<String\> | None 或非空 | 仅 Failed 时设置 |
| `created_at` | String | RFC3339 格式 | 任务创建时间 |

#### TransferStatus 状态机

```
     ┌──────────────────────────────────────┐
     │                                      │
     ▼                                      │
  Pending ──→ Transferring ──→ Completed    │
     │             │              │         │
     │             ▼              │         │
     │          Paused            │         │
     │             │              │         │
     │             ▼              │         │
     │         Pending (resume)   │         │
     │                          │         │
     └──→ Cancelled              │         │
         (任意状态)               │         │
                                 ▼         │
                              Failed        │
                                 │          │
                                 └──────────┘
```

有效的状态转换:

| 从 | 到 | 触发条件 |
|----|----|---------|
| Pending | Transferring | 队列调度，启动下载 |
| Transferring | Completed | 全部字节接收完成 |
| Transferring | Paused | 用户暂停 |
| Paused | Pending | 用户恢复 (重新排队) |
| Transferring | Failed | 网络中断/IO 错误 |
| Pending | Cancelled | 用户取消 |
| Transferring | Cancelled | 用户取消 |
| Paused | Cancelled | 用户取消 |
| Pending | Failed | 预留 (启动失败) |

### 9.2 性能基准

### 9.3 异常场景覆盖

| 场景 | 预期行为 |
|------|---------|
| mDNS 初始化失败 | 日志记录错误，应用继续运行，设备发现功能不可用但其他功能正常 |
| 远程设备突然断网 | 当前传输失败，任务标记 Failed；设备列表中标记 offline |
| 磁盘空间不足 | 下载时文件写入失败 → 任务标记 Failed，清理部分文件 |
| 共享目录被删除 | HTTP 服务在下次请求时返回 404 目录不存在 |
| 同时多设备连接同一台 | 每个连接独立处理，无锁竞争问题 (axum 异步处理) |
| 大目录 (100K+ inodes) | 列出时可能超时，但不会崩溃 (已有 10 秒超时) |
| 端口绑定失败 | 日志记录，应用继续运行，HTTP 服务不可用 |
| 路径遍历攻击 `../../../etc/passwd` | `resolve_safe_path` 捕获 → 返回 403 |
| 重复设备发现 (mDNS 重复事件) | 前端去重 (以 id 为 key) |
| 传入无效的 TransferTask id | 返回 "任务不存在" 错误 |
| WebSocket 连接中断 | 客户端自动重连 (未来版本) |
| 设置文件损坏 | 重置为默认设置，日志记录警告 |
