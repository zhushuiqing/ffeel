# 本地测试技能

> 一键启动桌面客户端和网页端，人工逐项测试。关于测试用例：按下面列出步骤逐一操作即可。

---

## 启动测试环境

```bash
# 终端 1：启动 Tauri 桌面客户端（含后端服务 + 前端热更新）
npm run tauri dev
```

**启动后将打印：**
```
HTTPS 服务启动在端口: xxxxx (TLS 1.3)
```

记录端口号，构造网页端地址：
```
https://localhost:{端口}
或
https://{局域网IP}:{端口}
```

> 桌面端打开后，在**Settings 页面**可查看当前服务地址和端口。

## 测试清单

### 1. 配对功能

| 步骤 | 预期 |
|------|------|
| 桌面端 → Devices 页，点击「显示配对码」 | 弹出配对码二维码 |
| 另一台设备浏览器打开网页端，输入配对码 | 配对成功，双方设备列表更新 |
| 桌面端 Settings 页「旋转配对码」 | 配对码刷新，旧码失效 |
| 桌面端 Devices 页「取消配对」某设备 | 该设备从信任列表移除 |

> 配对码有效期可在 `security/pairing.rs` 中调整。配对后信任关系永久保存。

### 2. 设备发现（Devices 页）

| 步骤 | 预期 |
|------|------|
| 启动桌面客户端 | 自动开始 mDNS 发现 |
| 同局域网另一台设备启动 ffeel | 自动出现在设备列表 |
| 设备离线 | 设备卡片显示「离线」状态 |

### 3. 文件浏览（FileBrowser 页）

| 步骤 | 预期 |
|------|------|
| 选择一个在线设备 | 显示该设备的共享目录 |
| 点击目录进入子文件夹 | 正确列出文件列表 |
| 搜索框输入文件名关键词 | 筛选匹配文件 |
| 翻页/滚动加载 | 分页正常 |

### 4. 文件传输（Transfers 页）

| 步骤 | 预期 |
|------|------|
| 在 FileBrowser 点击文件「下载」| 开始传输，Transfers 页显示进度条 |
| 传输中点击「暂停」| 进度暂停 |
| 点击「继续」| 从断点恢复 |
| 点击「取消」| 传输终止，文件清理 |
| 下载完成后 | 文件出现在本地下载目录 |
| 上传文件 | 另一台设备浏览器端可看到上传的文件 |

### 5. 聊天功能（Friends 页）

| 步骤 | 预期 |
|------|------|
| 桌面端 Friends 页 → 点击一个好友 | 进入聊天界面 |
| 输入文字 → 回车发送 | 消息显示在聊天气泡中 |
| 点击 📎 附件 → 选择文件发送 | 接收方文件卡片显示进度，下载后双击可打开 |
| 点击 😊 表情 → 选择表情 | 表情插入输入框 |
| 发送图片文件 | 聊天中显示缩略图，点击可放大预览 |
| 另一台设备浏览器端发送消息 | 桌面端实时显示（无需刷新） |

### 6. 网页端（Web UI）

| 步骤 | 预期 |
|------|------|
| 浏览器打开 `https://localhost:{端口}` | 加载 web_ui.html |
| 输入配对码 | 配对成功后进入主界面 |
| 设备列表 → 文件浏览/下载 | 功能正常 |
| 聊天 → 发送文字/文件/表情 | 与桌面端行为一致 |
| 顶部在线状态点 | 其他设备上下线实时更新 |

### 7. 跨平台互传

| 步骤 | 预期 |
|------|------|
| Mac 桌面 ←→ Windows 桌面 | 发现、配对、传输均正常 |
| 桌面端 ←→ 浏览器 | 文件互相传输正常 |
| 多设备同时在线 | 均可相互发现和通信 |

## 配套验证命令

测试过程中可附带运行后端测试：
```bash
# 终端 2：运行自动化测试
cd src-tauri && cargo test
```

打开浏览器开发者工具（F12）查看：
- Console：WebSocket 连接状态、消息通信日志
- Network：API 请求/响应

## 常见问题

| 问题 | 排查 |
|------|------|
| 桌面端连不上网页端 | 检查端口一致、防火墙、局域网互通 |
| 配对失败 | 确认配对码未过期，检查两个设备时间同步 |
| 文件传输慢 | 检查 `rate_limiter.rs` 限速配置 |
| 网页端消息不刷新 | 检查 WebSocket 连接状态（浏览器 F12 → Console） |
